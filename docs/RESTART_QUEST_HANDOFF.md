# Re:Start Quest 인수인계 문서

> 현재 문서는 아이디어와 배경을 보존하기 위한 인수인계 자료다. MVP 구현 계약은 `docs/MVP_DESIGN.md`와 `docs/BACKEND_API_CONTRACT.md`를 우선하며, 이 문서의 확장 기능 목록은 별도 work item이 생기기 전까지 구현 범위로 보지 않는다.

## 1. 이 문서의 목적

이 문서는 오케스트레이터 세션에서 개인 포트폴리오 프로젝트를 시작하기 위한 인수인계 자료다.

현재 목표는 이미 만들어둔 오케스트레이터를 실제 프로젝트 운영에 처음 적용해보고, 그 결과물로 포트폴리오와 이력서에 넣을 수 있는 완성도 있는 개인 프로젝트를 만드는 것이다.

프로젝트 주제는 **쉬었음 청년/취업 공백 청년을 위한 AI 자립 퀘스트 코치**다.

---

## 2. 아이디어가 나온 흐름

처음에는 K-디지털 트레이닝 해커톤 주제를 고민하면서 여러 아이디어를 검토했다.

해커톤 주제는 **"AI와 함께, 인간의 가능성을 확장하다"**였고, 여기서 중요한 방향은 AI가 사람을 대체하는 것이 아니라 사람이 더 잘 판단하고, 행동하고, 성장하도록 돕는 서비스라고 봤다.

검토했던 아이디어들은 다음과 같았다.

- 커뮤니티 도배/꾸준글/어그로 패턴 탐지 AI
- 공공문서 쉬운말 변환 및 행동 가이드 AI
- 보이스피싱 리허설 AI
- 청년 고립 탈출 첫 행동 코치
- 요양보호사 돌봄기록/인수인계 AI

이 과정에서 **청년 고립 탈출 첫 행동 코치** 아이디어를 보다가, 쉬었음 청년이나 취업 공백 청년에게도 비슷한 문제가 있다는 생각이 나왔다.

취업 공백 청년은 단순히 이력서 작성법을 몰라서 멈춰 있는 것이 아니라, 공백이 길어질수록 어디서부터 다시 시작해야 할지 막막해지고, 큰 목표 앞에서 작은 행동을 시작하기 어려워지는 경우가 많다.

그래서 기존 취업 AI처럼 이력서 작성이나 면접 연습 기능 하나만 제공하는 것이 아니라, **공고 탐색, 이력서 점검, 모의면접, 학습, 정부지원정책 확인을 "오늘 할 수 있는 작은 퀘스트"로 쪼개주는 서비스**를 생각하게 됐다.

---

## 3. 현재 내 상황

현재 나는 오케스트레이터를 만들어둔 상태지만, 실제 프로젝트에 연결해서 운영해본 적은 없다.

이번 프로젝트는 오케스트레이터를 처음으로 실전 운영해보는 테스트 프로젝트이자, 동시에 포트폴리오용 개인 프로젝트로 만들 계획이다.

즉, 목적은 두 가지다.

1. 오케스트레이터가 실제 개발 프로젝트를 잘 분해하고 운영할 수 있는지 검증한다.
2. 취업용 포트폴리오에 넣을 수 있는 AI 웹 서비스를 완성한다.

개발 방향은 개인 프로젝트 기준으로 현실적인 MVP부터 만든다.

처음부터 모든 기능을 다 만들기보다, 서비스의 차별점이 드러나는 핵심 흐름을 먼저 완성한다.

## 4. 프로젝트 한 줄 정의

## 6. 해결하려는 문제

취업 공백 청년에게 필요한 것은 단순한 정보 제공이 아니다.

사람인, 알바몬, 고용24, 정부지원정책, 이력서 작성 도구, 모의면접 도구는 이미 존재한다. 문제는 이런 기능들이 흩어져 있고, 사용자가 무엇을 먼저 해야 할지 모른다는 점이다.

핵심 문제는 다음과 같다.

- 취업 공백이 길어질수록 시작 자체가 어려워진다.
- 큰 목표는 부담스럽고, 작은 행동으로 쪼개진 계획이 필요하다.
- 기존 취업 AI는 이력서, 면접, 공고 추천처럼 기능이 분리되어 있다.
- 실패했을 때 다시 시작할 수 있게 재설계해주는 구조가 부족하다.
- 정부지원정책은 존재하지만 본인에게 어떤 제도를 확인해야 하는지 찾기 어렵다.

이 프로젝트는 취업을 대신해주는 AI가 아니라, **취업 준비를 다시 시작하게 만드는 실행 보조 AI**를 목표로 한다.

---

## 7. 핵심 차별점

기존 서비스와 차별화되는 지점은 기능 자체보다 흐름이다.

기존 서비스:

- 이력서 AI: 문장 작성/첨삭
- 면접 AI: 질문과 답변 피드백
- 채용 플랫폼: 공고 추천
- 정부 사이트: 정책 안내

이 프로젝트:

- 위 기능들을 사용자의 현재 상태에 맞는 **일일 퀘스트**로 연결한다.
- 실패하면 더 작은 행동으로 다시 쪼갠다.
- 취업 결과가 아니라 **실행 지속**을 중심에 둔다.
- 사용자를 평가하거나 감시하지 않고, 다시 움직일 수 있게 돕는다.

---

## 8. MVP 핵심 흐름

가장 먼저 구현해야 할 핵심 흐름은 아래 6단계다.

1. 사용자가 온보딩 정보를 입력한다.
2. 홈 화면에서 "오늘의 퀘스트 생성"을 누른다.
3. AI가 작은 취업 미션 3개를 생성한다.
4. 사용자가 미션을 완료 또는 실패 처리한다.
5. 실패하면 AI가 더 쉬운 미션으로 다시 쪼갠다.
6. 대시보드에 오늘의 진행 상태가 반영된다.

이 흐름만 제대로 구현해도 서비스의 핵심 차별점이 드러난다.

---

## 9. 주요 기능

### 9.1 온보딩

사용자가 처음 접속했을 때 현재 상태를 입력한다.

예상 입력값:

- 거주 지역
- 희망 직무
- 희망 근무 형태
- 취업 공백 기간
- 현재 이력서 보유 여부
- 면접 경험 여부
- 관심 분야
- 오늘 가능한 에너지 수준: 낮음 / 보통 / 높음

이 정보는 사용자를 평가하기 위한 것이 아니라, 미션 난이도와 방향을 조절하기 위한 데이터다.

### 9.2 오늘의 취업 퀘스트

AI가 사용자의 상태에 맞춰 오늘 할 수 있는 작은 미션을 만든다.

예시:

- 관심 직무 공고 3개 저장하기
- 이력서 제목 한 줄 수정하기
- 자기소개서 첫 문장 작성하기
- 모의면접 질문 3개 답변하기
- 고용24에서 확인할 정책 1개 찾기
- 희망 직무 관련 개념 20분 복습하기

퀘스트는 10~30분 안에 끝낼 수 있을 정도로 작아야 한다.

### 9.3 실패 후 재설계

사용자가 미션을 실패했을 때, AI가 더 쉬운 단계로 다시 쪼갠다.

예시:

- 기존 미션: 자기소개서 항목 작성하기
- 재설계 미션:
  - 자기소개서 문항만 읽기
  - 관련 경험 3개만 적기
  - 첫 문장만 작성하기

이 기능은 서비스의 핵심 차별점이다.

### 9.4 이력서/자기소개서 점검

사용자가 이력서나 자기소개서 내용을 입력하면 AI가 피드백한다.

기능:

- 문장 다듬기
- 직무와 맞는 표현 추천
- 부족한 경험을 찾기 위한 질문 생성
- 과장 표현 경고
- 오늘 수정할 항목 1~3개 추천

전체를 대신 써주는 방향보다, 사용자가 직접 개선할 수 있게 작은 수정 퀘스트로 연결한다.

### 9.5 모의면접

희망 직무와 이력서 내용을 바탕으로 면접 질문을 생성한다.

기능:

- 직무별 예상 질문 생성
- 공백 기간 설명 연습
- 답변 입력 후 피드백
- STAR 방식 구조화
- 다음 연습 미션 생성

### 9.6 공고 저장 및 지원 상태 관리

초기 MVP에서는 실제 크롤링 없이 더미 공고 또는 사용자가 직접 등록한 공고로 시작한다.

기능:

- 공고 저장
- 관심 / 준비 중 / 지원 완료 / 면접 예정 / 탈락 / 합격 상태 관리
- 공고별 준비 미션 생성
- 마감일 리마인드

향후 사람인, 알바몬, 고용24 연동은 확장 기능으로 둔다.

### 9.7 정부지원정책 안내

초기에는 정책 몇 개를 직접 DB에 등록해서 사용한다.

예시:

- 국민취업지원제도
- 청년도전지원사업
- K-디지털 트레이닝
- 내일배움카드

주의할 점:

- 대상 여부를 확정 판정하지 않는다.
- "확인해볼 만한 정책"과 "신청 전 확인 항목" 중심으로 안내한다.

## 14. 오케스트레이터에게 맡길 첫 작업

오케스트레이터는 처음부터 코드를 바로 짜기보다, 먼저 프로젝트를 운영 가능한 단위로 쪼개야 한다.

첫 요청은 아래처럼 한다.

```text
쉬었음 청년/취업 공백 청년을 위한 Re:Start Quest 웹 서비스를 만들려고 한다.
목표는 개인 포트폴리오 프로젝트이며, React + Spring Boot 기반 MVP를 만든다.

우선 PM 역할로 다음을 정리해줘.
1. MVP 범위
2. 사용자 시나리오
3. 기능 우선순위
4. 화면 목록
5. 백엔드 도메인 모델
6. API 초안
7. 개발 일정
8. 포트폴리오에서 강조할 차별점
```

그다음 오케스트레이터가 역할을 나눠서 프론트, 백엔드, AI, 문서화 작업을 분배하도록 한다.

---

## 15. 포트폴리오에서 강조할 점

- 단순 취업 AI가 아니라 실행 지속 문제를 해결하는 서비스
- 이력서, 면접, 공고, 정책을 일일 퀘스트로 연결한 점
- 실패한 사용자를 비난하지 않고 더 작은 행동으로 재설계하는 UX
- LLM을 단순 답변 생성이 아니라 사용자 상태 기반 행동 설계에 활용한 점
- React, Spring Boot, DB 모델링, AI API 연동을 모두 보여줄 수 있는 구조

---

## 16. 주의할 점

- 정신건강 상담 서비스처럼 보이면 안 된다.
- 사용자를 감시하거나 취업 의지를 평가하는 서비스처럼 보이면 안 된다.
- 정부지원정책 대상 여부를 확정 판정하면 안 된다.
- 실제 채용 사이트 크롤링은 처음부터 목표로 하지 않는다.
- 이력서를 AI가 대신 써주는 서비스가 아니라, 사용자가 직접 실행하도록 돕는 서비스로 유지한다.

---

## 17. 최종 목표

최종적으로 만들고 싶은 것은 단순한 AI 기능 모음이 아니다.

목표는 **취업 공백 청년이 다시 움직일 수 있도록, 큰 취업 준비 과정을 작은 행동으로 바꿔주는 실행 중심 AI 서비스**다.

첫 MVP에서는 "온보딩 → 오늘의 퀘스트 생성 → 완료/실패 → 실패 시 재설계 → 대시보드 반영" 흐름을 완성하는 데 집중한다.
