# Re:Start Quest 제품 방향성

## 1. 한 줄 정의

취업 공백자가 구직 행동을 다시 시작할 수 있도록, AI가 큰 취업 준비 과정을 작은 실행 단위로 쪼개고 실패 후에도 다시 시작할 수 있게 재설계하는 서비스입니다.

## 2. 핵심 포지셔닝

이 프로젝트는 공고 추천, 이력서 AI, 모의면접 AI, 정책 추천을 모아놓은 취업 종합 앱으로 보이면 약합니다.

진짜 차별점은 아래 흐름입니다.

```text
사용자가 취업 준비를 시작하려고 함
-> AI가 오늘 할 수 있는 작은 행동으로 쪼갬
-> 사용자가 실패함
-> 실패 이유를 입력함
-> AI가 더 낮은 난이도의 행동으로 재설계함
-> 다시 실행할 수 있게 루틴을 이어줌
```

핵심은 취업 기능 제공이 아니라 **구직 행동 재진입 엔진**입니다.

## 3. 초기 타깃

- 취업 의지는 있지만 공백이 3개월 이상 있는 청년
- 이력서, 면접, 공고 탐색 중 어디서부터 시작할지 막막한 사람
- 심각한 정신건강 상담이 필요한 사용자가 아니라, 구직 루틴 재시작이 필요한 사용자
- 혼자 취업 준비를 지속하기 어려운 사람

정신건강 치료, 상담, 우울증 개선 서비스처럼 보이면 안 됩니다. 이 프로젝트는 상담 서비스가 아니라 취업 준비 실행 보조 도구입니다.

## 4. MVP 핵심 기능

MVP에서 가장 중요한 기능은 아래 3개입니다.

1. 오늘의 구직 퀘스트 생성
2. 실패 이유 기반 퀘스트 재설계
3. 실행 기록 대시보드

이력서 첨삭, 모의면접, 공고 저장, 정책 추천은 전부 이 핵심 흐름을 보조하는 기능입니다. 일정이 부족하면 공고 연동이나 정책 추천보다 실패 후 재설계를 먼저 만듭니다.

## 5. 실패 이유 분류

사용자가 퀘스트를 실패했을 때 단순히 실패 처리만 하면 안 됩니다. 왜 실패했는지를 선택하게 하고, 그 이유에 따라 AI가 다르게 재설계해야 합니다.

실패 이유 예시:

- 시간이 부족했다
- 너무 부담스러웠다
- 무엇을 써야 할지 몰랐다
- 자료가 없었다
- 자신감이 없었다
- 공고가 마음에 들지 않았다
- 오늘 컨디션이 낮았다

재설계 예시:

```text
기존 퀘스트:
자기소개서 지원동기 작성하기

실패 이유:
무엇을 써야 할지 몰랐다

재설계:
1. 지원 기업의 키워드 3개만 적기
2. 내가 가진 경험 2개만 적기
3. 지원동기 첫 문장만 작성하기
```

## 6. AI 출력 정책

LLM 응답은 자연어 본문이 아니라 JSON 구조로 받습니다.

예시:

```json
{
  "quests": [
    {
      "title": "관심 공고 3개 저장하기",
      "description": "희망 직무와 지역에 맞는 공고를 3개 찾아 저장합니다.",
      "category": "JOB_SEARCH",
      "difficulty": "EASY",
      "estimatedMinutes": 15,
      "completionCriteria": "공고 3개를 저장하면 완료",
      "fallbackQuest": "공고 사이트에 접속해서 직무 키워드 1개만 검색하기"
    }
  ]
}
```

이렇게 해야 프론트에서 카드로 보여주기 쉽고, DB 저장과 검증도 단순해집니다.

## 7. 데모 시나리오

포트폴리오 시연은 아래 흐름을 기준으로 합니다.

1. 사용자가 온보딩 입력
   - 취업 공백 8개월
   - 희망 직무: 백엔드 개발자
   - 이력서 있음
   - 면접 경험 적음
   - 오늘 에너지 낮음
2. AI가 오늘의 퀘스트 3개 생성
   - 공고 2개 저장하기
   - 이력서 프로젝트 설명 한 문단 수정하기
   - 공백 기간 답변 3문장 작성하기
3. 사용자가 이력서 수정 퀘스트 실패 처리
   - 실패 이유: 무엇을 써야 할지 몰랐다
4. AI가 더 쉬운 퀘스트로 재설계
   - 프로젝트에서 사용한 기술 3개만 적기
   - 맡은 역할 1문장만 적기
   - 문제 해결 경험 1개만 고르기
5. 대시보드에 오늘의 진행률, 실패 후 재설계 기록, 다음 행동이 표시됨

## 8. 피해야 할 방향

- 그냥 취업 정보 모음 앱처럼 만들지 않기
- 이력서 AI 첨삭 서비스처럼만 보이게 만들지 않기
- 정신건강 상담 서비스처럼 보이게 만들지 않기
- 사용자를 감시하거나 취업 의지를 평가하는 서비스처럼 만들지 않기
- 모든 기능을 한 번에 구현하려고 하지 않기

가장 중요한 것은 실패 후 더 작은 행동으로 재설계되는 경험입니다. 이 기능이 없으면 흔한 취업 AI이고, 이 기능이 있으면 Re:Start Quest가 됩니다.
